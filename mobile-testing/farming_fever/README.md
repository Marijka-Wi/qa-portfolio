\# QA Portfolio: Farming Fever Test Assignment



This is a QA test assignment I completed for the game \*\*Farming Fever\*\*.  

The goal was to explore the game, test its functionality, identify issues, and create a structured test report.



---



\## Testing Objective

\- Verify game functionality (menus, levels, resources, interactions)

\- Identify bugs and inconsistencies

\- Evaluate usability and user experience

\- Document test scenarios for future reference



---



\## Folder Structure

\- `/reports` – detailed test reports, including steps to reproduce bugs

\- `/bugs` – individual bug documentation with screenshots

\- `/screenshots` – visual evidence of issues or test steps

\- `/scenarios` – test scenarios and checklists

\- `/notes` – personal observations and recommendations



---



\## Example Test Scenarios

\### Functional Testing

\- Check that levels load correctly

\- Verify that resources (coins, crops, animals) update properly

\- Ensure buttons and menus respond as expected



\### Usability Testing

\- Evaluate game navigation for clarity and ease of use

\- Check tooltips, instructions, and hints

\- Verify game messages are understandable



\### Compatibility / Device Testing

\- Test on different screen resolutions

\- Check for mobile responsiveness if applicable



---



\## Bug Reports

All detailed bug reports are in the `/reports` folder.  

Individual bugs with reproduction steps and screenshots can be found in the `/bugs` folder.



---



\## How to Use

1\. Open the `/reports` folder to see detailed test results.

2\. Refer to `/bugs` for specific bug documentation.

3\. Check `/scenarios` for step-by-step test cases.

4\. Screenshots are in `/screenshots`.

5\. Notes and personal observations are in `/notes`.
